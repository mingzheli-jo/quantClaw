# QuantClaw - A股量化交易信号系统设计文档

> 日期: 2026-05-26
> 状态: 已批准
> 域名: quant.azhefuye.online

## 1. 项目概述

QuantClaw 是一个个人 A 股量化交易信号系统，通过全市场扫描和多因子打分，每日推荐买入/卖出候选股票，并通过飞书和 Web 看板通知用户手动执行交易。

### 1.1 核心定位

- **不是自动交易系统**：生成信号 → 通知用户 → 用户手动执行
- **面向短线交易**：持有周期 1-5 个交易日
- **稳健风格**：信号少但准确率高，每天最多推荐 3 只

### 1.2 用户画像

- 单用户（项目所有者）
- 起步资金 5W，后期可能追加到 10W
- 量化小白，需要系统解释每个信号的逻辑
- 部署在腾讯云 2核4G 服务器

### 1.3 A 股特殊规则约束

| 规则 | 影响 |
|------|------|
| T+1 | 今天买明天才能卖，回测以次日开盘价成交 |
| 涨跌幅限制 | 主板 ±10%，创业板/科创板 ±20% |
| 交易时间 | 9:30-11:30，13:00-15:00 |
| 无做空 | 只做多，只赚上涨的钱 |

## 3. 数据引擎

### 3.1 数据采集范围

| 数据类型 | 频率 | 用途 | 存储策略 |
|---------|------|------|---------|
| 全市场日 K 线 | 每日 15:30 | 技术指标计算 | 滚动保留近 250 个交易日 |
| 实时行情快照 | 盘中每 30 分钟 | 异动监控、持仓跟踪 | 仅保留当日 |
| 北向资金 | 每日 15:30 | 资金面因子 | 滚动 250 日 |
| 板块涨跌 | 每日 15:30 | 板块热度、轮动 | 滚动 250 日 |
| 个股资金流 | 每日 15:30 | 主力资金因子 | 滚动 60 日 |
| 涨跌停统计 | 每日 15:30 | 市场情绪 | 滚动 60 日 |
| 基本面（PE/PB/ROE） | 每周更新 | 过滤 + 辅助因子 | 最新快照 |

### 3.2 数据库表

**核心表：**

- `stock_daily` — 日 K 线（code, date, open, high, low, close, volume, amount）
- `stock_basic` — 股票基本信息（code, name, industry, market, list_date）
- `north_flow` — 北向资金（date, buy, sell, net）
- `sector_daily` — 板块行情（sector, date, change_pct, volume）
- `fund_flow` — 个股资金流（code, date, main_net, super_large_net...）
- `market_sentiment` — 市场情绪（date, up_count, down_count, limit_up, limit_down）

**信号与交易表：**

- `signal` — 策略信号（code, date, direction, score, reason, strategy）
- `position` — 持仓跟踪（code, buy_date, buy_price, status, stop_loss, take_profit）
- `trade_log` — 交易日志（code, date, action, price, reason, pnl）

**系统表：**

- `user` — 用户（单用户，登录用）
- `strategy_config` — 策略参数配置
- `backtest_result` — 回测结果
- `scheduler_log` — 调度日志

### 3.3 定时调度（交易日）

| 时间 | 任务 | 输出 |
|------|------|------|
| 08:30 | 盘前准备：检查交易日、刷新止盈止损价位 | 飞书"今日关注"推送 |
| 09:35 | 开盘快照：集合竞价结果、持仓高开/低开检测 | 异动推送（触发时） |
| 11:35 | 午盘快照：上午行情、持仓盈亏更新 | 触及止损则推送 |
| 15:05 | 收盘采集：全市场日 K 线、北向、板块、资金流入库 | - |
| 15:30 | 策略计算：过滤 → 打分 → 排名 → 信号生成 | 飞书盘后报告推送 |
| 20:00 | 数据维护：清理过期数据、每周一更新基本面 | - |

非交易日自动跳过。

### 3.4 数据采集容错

- 重试：失败后等待 30 秒重试，最多 3 次
- 分批：全市场 5000 只分 10 批采集，每批间隔 3 秒
- 校验：入库前检查数据完整性（无 NaN、日期正确、价格合理）
- 报警：连续失败超过 3 次 → 飞书推送错误通知

## 4. 策略引擎

### 4.1 选股流水线

```
全市场 ~5000 只
    ↓ 第一层：硬过滤（→ ~300-500 只）
    ↓ 第二层：多因子打分
    ↓ 第三层：集中度控制（→ TOP 3）
```

### 4.2 第一层：硬过滤

| 过滤条件 | 原因 |
|---------|------|
| ST / *ST | 退市风险 |
| 停牌 | 无法交易 |
| 上市不足 60 天 | 波动无规律 |
| 日均成交额 < 5000 万（20 日均值） | 流动性差 |
| 股价 > 50 元 | 5W 资金单只买不够 1 手 |
| 北交所 | 流动性和规则特殊 |
| 当日涨停 | 买不进 |
| 当日跌停 | 趋势破位 |

### 4.3 第二层：多因子打分

总分 = 技术面(40%) + 资金面(30%) + 动量面(20%) + 情绪面(10%)

**技术面（40 分满分）：**

| 指标 | 条件 | 得分 |
|------|------|------|
| 均线多头排列 | MA5 > MA10 > MA20 | 10 |
| MACD | DIF 上穿 DEA（金叉），且在零轴附近 | 8 |
| 布林带 | 价格突破中轨向上轨运行 | 7 |
| KDJ | K 值 20→80 区间金叉（非超买区） | 5 |
| 成交量 | 今日量 > 20 日均量 × 1.5 | 5 |
| 突破形态 | 收盘价创 20 日新高 | 5 |

**资金面（30 分满分）：**

| 指标 | 条件 | 得分 |
|------|------|------|
| 北向资金 | 近 3 日净买入 > 0 | 10 |
| 主力资金 | 今日主力净流入 > 0 | 10 |
| 超大单 | 超大单净买入占比 > 5% | 5 |
| 量比 | 量比 > 1.5 且 < 5 | 5 |

**动量面（20 分满分）：**

| 指标 | 条件 | 得分 |
|------|------|------|
| 5 日涨幅 | 3%-10% 区间 | 8 |
| 相对强弱 | 跑赢所属板块指数 | 7 |
| 突破前高 | 站上近 20 日最高收盘价 | 5 |

**情绪面（10 分满分）：**

| 指标 | 条件 | 得分 |
|------|------|------|
| 板块热度 | 所属板块当日涨幅排名前 20% | 5 |
| 市场情绪 | 涨停家数 > 跌停家数 × 2 | 3 |
| 板块资金 | 板块整体资金净流入 | 2 |

**信号阈值：** 总分 >= 65 分才进入候选池。

### 4.4 第三层：集中度控制

- 同板块最多选 1 只（分散风险）
- 与当前持仓不重复
- 最终输出 TOP 3

### 4.5 仓位管理

```
5W 资金:
├── 最多同时持有 2 只
├── 单只仓位 = 总资金 × 40%～50%（2W～2.5W/只）
├── 预留现金 >= 10%
│
买入规则:
├── 当前持仓 < 2 只 且 有候选 >= 65 分 → 允许买入
├── 当前持仓 = 2 只 → 不买入
└── 单只金额 = min(可用资金 × 50%, 2.5W)
```

### 4.6 卖出规则（按优先级）

| 优先级 | 规则 | 条件 | 动作 |
|--------|------|------|------|
| P1 | 止损 | 浮亏 >= 5% | 立即卖出信号 |
| P2 | 移动止盈 | 浮盈曾达 7%，从最高点回撤 3% | 卖出信号 |
| P3 | 固定止盈 | 浮盈 >= 12% | 建议止盈 |
| P4 | 时间止损 | 持有 >= 5 个交易日且未达止盈 | 卖出信号 |
| P5 | 技术破位 | 收盘跌破 MA10 | 卖出建议（非强制） |

### 4.7 策略可配置化

所有参数存数据库，Web 看板可调整：

```json

        "min_amount_20d": 50000000,
```

## 5. 通知系统

### 5.1 飞书 Webhook 机器人

使用飞书群组自定义机器人的 Webhook URL 发送消息卡片。

### 5.2 消息类型

| 类型 | 触发时间 | 内容 |
|------|---------|------|
| 盘前关注 | 08:30 | 持仓止盈止损价位提醒 + 关注列表 |
| 异动提醒 | 盘中 | 持仓高开/低开 > 3%、触及止损线 |
| 盘后报告 | 15:30 | 买入候选 TOP 3 + 持仓盈亏 + 市场温度 + 学习卡片 |
| 卖出信号 | 15:30 | 触发止盈/止损/时间止损的持仓 |
| 系统告警 | 异常时 | 数据采集失败、调度异常 |

### 5.3 盘后报告卡片结构

飞书 interactive message card 格式，包含：
- 市场概览：大盘指数涨跌、涨跌停家数、北向资金
- 市场温度：0-100 分（综合涨跌比、北向资金、涨停数等）
- 买入候选 TOP 3：评分、现价、买入区间、止损位、目标位、推荐理由
- 持仓跟踪：每只持仓的成本价、现价、盈亏比例、持有天数、操作建议
- 今日学习：一个技术指标的简明解释 + 结合当日实例
- 查看详情链接：跳转到 Web 看板

## 7. 回测引擎（P2）

### 7.1 回测流程

用户选择策略参数、时间范围（最近 1/3/6/12 个月）、初始资金（默认 5W）。

引擎逐个交易日执行：
1. 用当日数据运行完整选股流水线（和实盘完全相同的逻辑）
2. 检查虚拟持仓是否触发卖出规则
3. 有空仓位且有买入候选时，以**次日开盘价**模拟买入（模拟 T+1 真实情况）
4. 记录当日净值、持仓、交易

### 7.2 统计指标

| 指标 | 含义 | 健康基准 |
|------|------|---------|
| 年化收益率 | 折算 1 年收益 | > 15% |
| 最大回撤 | 最高点到最低点的最大亏损 | < 15% |
| 夏普比率 | 风险调整后收益（无风险利率 2%） | > 1.0 |
| 胜率 | 盈利笔数 / 总笔数 | > 55% |
| 盈亏比 | 平均盈利 / 平均亏损 | > 1.5 |
| 平均持有天数 | 每笔交易平均持有时间 | 2-5 天 |
| 交易频率 | 月均交易笔数 | 8-15 笔 |

### 7.3 手续费模拟

- 买入：佣金万分之 2.5（最低 5 元）
- 卖出：佣金万分之 2.5（最低 5 元）+ 印花税千分之 1

## 8. 学习模块

### 8.1 指标图鉴（内置 10 个）

MA 均线、MACD、KDJ、RSI、布林带、成交量、量比、北向资金、主力资金流、板块轮动。

每个指标包含：名称 + 一句话解释、简化计算公式、买入/卖出形态图示、使用场景、常见陷阱、在 QuantClaw 打分中的权重。

### 8.2 每日一学

- 10 个指标循环，每天讲一个
- 结合当日实际推荐案例
- 飞书推送精简版（3-4 行），Web 看板完整版（配图 + 历史案例）

### 8.3 历史信号复盘

每笔交易结束后自动生成复盘：买入时打分 + 理由、持有期间走势、卖出原因、最终盈亏、信号质量评价。

积累后在 /learn 页面展示：近 30 天信号准确率走势、各类信号胜率对比、用户跟单执行率。
